# VidLens（映知）首轮宣传制作指南

> 目标：面向 Go / 后端开发者，通过 B 站 3–5 分钟演示视频和小红书图文，为 GitHub 仓库获得第一批真实关注和 Star，并为简历建立可验证的公开项目展示。

## 1. 内容定位

### 受众

- Go / 后端开发者
- 对 AI 应用工程、异步任务、大文件上传和 RAG 感兴趣的开发者
- 寻找开源项目或简历项目参考的程序员

### 内容风格

采用“轻微整活 + 正经演示”的混合路线：

1. 前 5–12 秒使用曼波 AI 配音制造钩子。
2. 随后切换到剪映自然男声，展示真实产品闭环。
3. 先证明项目能运行，再解释它为什么不只是一次 AI API 调用。
4. 诚实说明这是个人开源项目，不包装成生产级商业平台。
5. B 站和小红书都尽量直接提供 GitHub 入口。

### 第一条视频只传达两个印象

- 这个项目真的可以运行。
- 作者处理了不少真实的后端工程问题。

## 2. 录制工具：OBS Studio

推荐下载 **OBS Studio**，不要下载 Streamlabs、直播伴侣或名称相似的第三方软件。

- 官方网站：<https://obsproject.com/>
- Windows 选择：`Windows Installer`
- 软件名称：`OBS Studio`
- OBS 免费、开源，不需要登录账号。

如果暂时不想安装 OBS，也可以使用剪映自带的录屏功能；但 OBS 的画质和稳定性更好，适合录网页和代码。

### OBS 首次配置

1. 打开 OBS，在“来源”区域点击 `+`。
2. 选择“窗口采集”，只捕获浏览器窗口；需要录终端或 VS Code 时再建立单独的窗口采集。
3. “设置 → 视频”：
   - 基础画布：`1920 × 1080`
   - 输出分辨率：`1920 × 1080`
   - 帧率：`30 FPS`
4. “设置 → 输出 → 录像”：
   - 录像格式：`MKV`
   - 质量：高质量、中等文件大小
5. 录完后使用“文件 → 录像转封装”，把 MKV 转成 MP4，再导入剪映。
6. 默认关闭桌面音频和麦克风，只录画面；只有确实需要保留原视频声音时，才单独开启桌面音频并确认版权。录制 10 秒样片后，在 OBS 混音器确认两路音频没有电平活动。

录制前先做 10 秒测试，检查：

- 画面是否清晰；
- 页面有没有被裁掉；
- 鼠标能否看清；
- 是否录入了通知声；
- 导出的 MP4 能否正常导入剪映。

## 3. 演示素材准备

选择一段 10–20 分钟、结构清楚的技术教程，最好与 Go、Kafka、Redis、RAG 等主题有关。避免版权风险较高的影视、综艺内容。

提前准备两个任务：

- **演示任务 A**：正式使用，已完成转写、摘要和索引，并提前验证问答效果。
- **备用任务 B**：发生模型错误或演示数据异常时立即替换。

视频里展示上传过程，但转写、摘要和问答直接使用已经处理完成的任务，不要让观众现场等待。上传镜头、摘要和问答必须使用同一个演示任务 A，文件名、封面和时长保持一致；备用任务 B 只在正式演示失败时替换整套流程。

### 提前测试的问题

1. 总结型：`这段视频主要讲了哪几个知识点？`
2. 细节型：围绕视频中确实出现的技术问题提问。
3. 备用问题：选择一个能稳定命中具体引用片段的问题。

## 4. B 站视频结构与逐段旁白

目标时长约 4 分钟 20 秒。

### 00:00–00:12：开头钩子

**画面**

- 展示十几分钟的技术视频并快速拖动进度条。
- 快切到 VidLens 摘要、视频问答和引用结果。
- 使用大字幕突出“视频太长”和“直接问它”。

**曼波 AI 配音**

> 不是哥们，一个十几分钟的技术视频，你真准备从头看到尾啊？  
> 我直接把视频丢给 AI，然后问它里面到底讲了什么。

**大字幕**

> 视频太长，不想看完？  
> 那就直接问它。

曼波音色只使用 12 秒以内，后面切换为自然男声。

### 00:12–00:35：介绍项目

**画面**

- GitHub README 顶部。
- VidLens 工作台。
- 角落显示 `Go / Kafka / Redis / MinIO / Milvus`。

**配音**

> 这是我用 Go 做的开源项目 VidLens，中文名叫映知。  
> 它可以处理视频转写和 AI 摘要，还能把视频内容建立成知识库，让用户直接针对视频进行提问，并返回相关的引用片段。

避免使用“准确理解所有视频”“完全替代看视频”等夸张表述。

### 00:35–01:25：上传和处理

**画面**

1. 选择技术教程视频。
2. 展示上传进度。
3. 快切到已经处理完成的同一视频。
4. 展示转写和摘要。

**配音**

> 先上传一段技术教程。对于比较大的文件，前端会进行分片上传；如果上传意外中断，也可以从已有进度继续，不需要重新传完整文件。  
> 上传完成后，后端会异步执行音频提取、分段转写、摘要生成和检索索引。为了避免视频处理时间影响普通请求，这些步骤不会一直阻塞在上传接口里。  
> 处理完成以后，可以查看完整转写，以及模型生成的视频摘要。

**画面提示**

```text
演示过程经过剪辑加速
分片上传 → ASR 转写 → 摘要 → RAG 索引
```

### 01:25–02:10：视频问答

**画面**

- 输入总结型问题。
- 展示回答和引用片段。
- 对引用内容做轻微放大或黄色描边。
- 再展示一个具体技术问题。

**配音**

> 除了摘要，我还可以直接针对视频内容进行提问。  
> 比如让它列出视频里的主要知识点，或者追问某个具体技术细节。  
> 回答不会只给出一段无法核对的文本，而是同时保留检索到的引用片段，方便判断答案是否来自原视频。

如果当前界面不支持点击时间跳转，只说“引用片段”，不要承诺时间跳转。

### 02:10–03:35：为什么不只是调用一次 AI API

**画面**

- 展示 README 架构图。
- 根据配音依次高亮 MinIO、Kafka、MySQL、Milvus/BM25 和 Redis。
- 穿插 2–3 个源码特写，每个不超过 5 秒。

**配音**

> 表面上看，它只是上传视频然后调用 AI，但真正麻烦的是视频文件大、处理时间长，而且中间任何一个外部服务都可能失败。  
> 文件部分使用 MinIO 保存，并通过分片上传和断点续传减少中断后的重复传输。  
> 转写、摘要和索引等长耗时步骤通过 Kafka 异步调度，MySQL 记录每个阶段的状态。任务失败后，会根据错误类型决定重试还是快速失败，避免所有错误都无脑重跑。  
> 在视频问答部分，我以 ASR 转写作为知识源，同时进行向量和关键词检索，再使用 RRF 融合结果。关键词检索用于补充向量检索对技术名词、缩写和代码词汇的召回，再通过 RRF 融合两路结果。  
> 项目还使用 Redis 处理上传状态、高成本接口限流和部分任务协调，并提供结构化日志与 Prometheus 指标，方便排查异步处理链路。

### 03:35–04:05：项目定位

**画面**

- 展示目录结构和测试文件。
- 展示一次真实的 `go test ./...` 结果。
- 回到 README。

**配音**

> VidLens 目前主要是我用于学习和验证后端工程方案的个人开源项目，还不是生产环境的视频平台。  
> 不过从大文件上传、异步任务、失败恢复，到视频检索问答，主要链路都可以在本地实际运行。  
> 我也会继续完善测试、部署体验和检索效果。

### 04:05–04:25：引导 Star

**画面**

- GitHub 仓库首页。
- 鼠标移动到 Star 按钮附近。
- 显示仓库地址以及“简介和置顶评论可见”。

**配音**

> 项目已经开源，地址放在视频简介和置顶评论。  
> 如果这个项目对你有一点启发，欢迎点个 Star；如果你对实现方案有建议，也欢迎直接提 Issue。  
> 我之后还会单独分享里面的分片上传、任务重试和 RAG 检索实现。

## 6. 剪映制作流程

### 工程设置

- 比例：`16:9`
- 分辨率：`1080p`
- 帧率：`30 FPS`
- 不套用花哨热门模板。

### 正确顺序

1. 按段落生成 AI 配音。
2. 把配音铺到时间线。
3. 根据配音节奏铺录屏画面。
4. 删除页面等待、鼠标找按钮等无效片段。
5. 使用直接切镜或 0.2 秒淡化。
6. 所有配音确定后，再生成自动字幕。
7. 人工校对技术名词。
8. 最后添加轻量背景音乐和局部高亮。

### 剪映首次操作路径

如果第一次使用剪映，按下面的菜单顺序操作：

1. 打开剪映桌面版，点击“开始创作”，画布比例选择 `16:9`。
2. 点击“导入”，导入 OBS 转成的 MP4、截图和背景音乐。
3. 先在时间线中放入录屏素材，保存工程。
4. 点击“文本”，新建文本框，按旁白段落粘贴文案。
5. 选中文本，使用“文本朗读/智能配音”，选择剪映内明确允许公开发布的音色，分别生成每一段音频。
6. 把生成的音频放入主时间线；根据音频长度剪切和排列录屏。
7. 配音确定后，使用“文本 → 智能字幕/识别字幕”，语言选择“中文普通话”，识别配音轨。
8. 在字幕轨道中全选字幕，统一修改字体、字号、描边和位置，再逐句检查技术名词。
9. 导出时选择 `1080p / 30 FPS / H.264`；码率按“推荐/高质量”即可，若显示自定义码率可使用 `12–20 Mbps`。

### AI 配音

仅使用剪映等正规平台提供且允许公开发布的音色，不使用来源不明的声音模型，也不仿冒具体自然人的声音。

分段生成并命名：

```text
01-hook-manbo
02-project-intro
03-upload
04-rag-chat
05-architecture
06-honest-positioning
07-call-to-action
```

- 曼波只用于开头。
- 正文统一使用一个自然男声。
- 语速约为默认的 `0.95–1.05` 倍。
- 技术名词读错时，拆成单独句子重新生成。
- `Milvus` 如果总读错，可以改说“向量数据库”。

### 字幕

- 清晰的无衬线字体；
- 1080p 下字号约 `42–52`；
- 白字、黑色描边；
- 每行 16–20 个汉字，最多两行；
- 位于底部安全区；
- 只对少量技术关键词使用黄色或蓝色强调。

必须人工检查：

```text
Go / Kafka / Redis / MinIO / Milvus
RAG / ASR / BM25 / RRF / Prometheus
```

### 音乐

- 使用无歌词、低存在感的科技感音乐；
- 背景音乐保持在配音音量的约 `8%–15%`；
- 不使用版权来源不明的热门歌曲；
- 没有合适音乐时可以不加。

### 导出

- `1080p`
- `30 FPS`
- `H.264`
- `12–20 Mbps`

导出后戴耳机完整看一遍，并在手机上检查字幕和代码是否可读。

1. **视频太长不想看？我用 Go 做了一个 AI 视频问答项目**
2. **我把视频做成了知识库：Go + RAG 开源项目演示**
3. **不只是调 API：我用 Go 做了一个 AI 视频处理后端**

5. **一个能“问视频”的 Go 项目：转写、总结和引用式问答**

## 10. 一个周末的执行安排

### 周六上午（约 2 小时）

- 安装并测试 OBS；
- 选择技术视频；
- 跑完演示任务 A 和备用任务 B；
- 验证三个问答问题；
- 清理演示账号和页面。

### 周六下午（约 2–3 小时）

- 按镜头清单分段录屏；
- 每个关键画面录两遍；
- 命名和整理素材。

### 周日上午（约 2 小时）

- 在剪映分段生成 AI 配音；
- 按配音铺画面；
- 完成粗剪。

### 周日下午（约 2–3 小时）

- 生成并校对字幕；
- 添加局部高亮和轻量音乐；
- 制作 B 站封面；
- 导出并用电脑、手机各检查一遍；
- 发布 B 站，再制作小红书图文。
